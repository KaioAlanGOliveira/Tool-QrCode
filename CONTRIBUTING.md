# Contributing to Tool-QrCode

Thank you for your interest in contributing to Tool-QrCode! We welcome contributions from everyone. Please follow these guidelines to ensure a smooth contribution process.

## Contribution Guidelines
1. **Fork the repository**: Create your own copy of the repository to work on.
2. **Clone your fork**: Clone your forked repository to your local machine to begin working on your changes.
3. **Create a new branch**: Make sure to create a new branch for your changes. Use a descriptive name that reflects the purpose of your contribution (e.g., `feature/add-new-feature`).
4. **Make your changes**: Implement your changes, ensuring that your code adheres to our coding standards.
5. **Test your changes**: Run the tests to make sure everything works correctly.
6. **Commit your changes**: Use clear and descriptive commit messages. Follow the commit message standards outlined below.
7. **Push your changes**: Push your changes to your forked repository.
8. **Open a Pull Request**: Submit a pull request to the main repository for review.

## Commit Message Standards
All commit messages should follow these guidelines:
- Start with a brief summary of the changes (50 characters or less).
- Use the imperative mood (e.g., "Add feature" instead of "Added feature").
- Provide a more detailed explanation if necessary in the following lines.
- Reference any relevant issues or pull requests.

Example of a commit message:
```
Add new feature to generate QR codes

This feature allows users to generate QR codes based on user inputs.
Closes #24
```

Thank you for helping us improve Tool-QrCode!
